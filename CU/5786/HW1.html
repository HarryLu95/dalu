Question 1: The picture mishap¶

The back story: You took some photos of smiley faces a while ago and knowing about the PCA algorithm, compressed these photos. The original photos were of size $105 \times 105$ grey-scale images represented as $d = 11025$ dimensional vectors (remember the smiley faces from class!). The compressed photos were $K = 20$ dimensional each along with the projection matrix $W$ of size $11025 \times 20$ and the mean vector $\mu$ of size $11025 \times 1$.

Accidentally one night, you deleted all but one of the compressed version of the photos and all you have now is the projection matrix W, the mean vector µ and the compressed 20 dimensional vector of just the first image say $y_1$. Fortunately for you, one of your friends has printed copies of these photos. However as luck would have it, these photos were all stained with an identical stain on all of them (coffee spill over the stack of photos). To make things worse, your friend takes a picture of all these photos at an angle (in this assignment for my ease a $90$ degree angle but in general this is some unknown angle of rotation) and emails these photos over to you. So now you have these $28$ images all rotated and all with identical stain on them. Can you use your knowledge of PCA to reconstruct your favorite smiley photos?

For your convenience, below we load:

  1. the projection matrix (of size $11025 \times 20$) you used into variable "Wgood"
  2. the mean face you had originally computed in the vectorized format into variable "mugood"
  3. The low dimensional ($20$ dimensional) representation of the first image you luckily had on you in variable "Ygood1"

Hint:¶

Adding the same stain on all the images can be viewed as images all being translated by adding the stain

So you can view each $\mathbf{x}_t$ as now changes to $\mathbf{x}_t + \mathrm{stain}$.

Think of how this translation would affect the new projection matrix $W$, and the low dimensional $\mathbf{y}_t$'s

Next, the rotation of the image can be thought of as exaclty that, rotating the instances by some rotation matrix $R$. So you can think of rotation + stain as changing $\mathbf{x}_t$ to $R (\mathbf{x}_t + \mathrm{stain})$ where the rotation matrix $R$ is a valid rotation matrix with $R^\top R = R R^\top = I$. If we rotate data point, how does it affect new projection matrix $W$? How does it affect magnitude of coordinates of $\mathbf{y}_t$?

One other hint, and the only reason we need the low dimensional version of the first image: if $x$ is an eigen vector of a matrix $M$ then so is $-x$. Its like saying we can put the arrow left to right or right to left. But notice that if we flip the sign of a column of $W$, then corresponding coordinate of $\mathbf{y}_t$ is also flipped. You will need this observation and the low dimensional representation of the first image to get the right reconstruction.

def Q1_solution(Wgood, mugood, Ygood1,X):
    Xhat = np.zeros((28,11025))
    
    def Rotate_90_To_Right(d):
        matrix = np.zeros((d**2,d**2))
        for i in range(d):
            for j in range(d):
                matrix[i+j*d, (i+1)*d-j-1]=1
        return matrix
    
    def Rotate_90_To_Left(d):
        matrix = np.zeros((d**2,d**2))
        for i in range(d):
            for j in range(d):
                matrix[(i+1)*d-j-1, i+j*d]=1
        return matrix

    R = Rotate_90_To_Right(105)

    X_rot = X.dot(R)
    
    mu_bad = np.mean(X_rot, axis = 0)
    Sigma_bad = np.cov(X_rot.T)
    K = 20
    E,Wbad = eigs(Sigma_bad, k = K, which = "LM")
    Wbad = np.real(Wbad)
    Ybad = (X_rot - mu_bad).dot(Wbad)
    
    Xhat = Ybad.dot(Wbad.T) + mugood
    
    return Xhat

Question 2: PCA Vs Random Projections¶

Your goal in this question is to generate two data sets consisting of $100$ points in $1000$-dimensions each. For each $d$-dimensional point $\mathbf{x}_t$ in the two data sets, ensure that their norm (distance to $0$) is exactly 1. We shall perform PCA and random projections on both the data sets to K = 20 dimensions. The data sets should be such that on the first one PCA outperforms random projection by a large margin and in the second, random projection outperforms PCA. To evaluate our projections we shall use the following metric on how well the projections preserve average inter point distances:

$$ \mathrm{Err}(\mathbf{y}_1,\ldots, \mathbf{y}_n) = \frac{1}{n(n − 1)/2} \sum_{i=1}^n \sum_{j=i+1}^n \left|\ \left\|\mathbf{y}_j − \mathbf{y}_i\right\| − \left\|\mathbf{x}_j − \mathbf{x}_i\right\|\ \right| $$

We shall pick $K = 20$ and perform PCA and random projections on both the data sets. Your task in this problem is to create the two data sets, one in which PCA beats random projections when one measures the error term above and another one where random projection beats PCA.

You are not allowed to use random number generators to generate either of the datasets. Your code should deterministically generate the data.
